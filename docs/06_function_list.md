1. 本書の目的

本書は、Research Session Hub のMVPで実装対象とする機能を整理し、各機能の目的、対象ユーザー、概要、優先度、MVP実装範囲を明確化することを目的とする。

本MVPは3日で動作するプロトタイプの構築を前提とするため、本書では機能を厳選し、投稿、確認、修正、公開、議論という中核ワークフローを成立させる機能に集中する。

2. 機能一覧の分類

本書では機能を以下の分類で整理する。

・共通機能
・認証機能
・研究発表者機能
・閲覧者機能
・事務局機能
・論文公開機能
・コメント、議論機能
・ファイル管理機能
・システム制御機能
・MVP対象外機能

3. 優先度定義

Must
MVP成立に必須の機能

Should
MVP内で可能なら入れるべき機能
ただしMust優先

Could
将来拡張候補
MVPでは原則実装しない

15. MVP簡略化方針

Research Session Hub のMVPでは、以下のように機能を簡略化する。

・投稿編集は新規投稿画面と統合する
・投稿詳細確認は省略可能とする
・ユーザー管理は極小機能にする
・コメント返信は実装しない
・通知は実装しない
・公開処理はまず手動でもよい
・公開管理とコメント管理は事務局投稿詳細に統合する
・検索は簡易版に留める

17. 最終判断

Research Session Hub のMVP機能は、研究発表者による投稿、事務局による確認と差し戻し、会員限定公開、コメント議論、後日一般公開という一連の流れを成立させることに集中する。

本書の最終方針は以下とする。

・コアワークフローに関係ない機能はMVPから外す
・研究発表者機能は提出と再提出を最優先にする
・事務局機能は一覧確認、差し戻し、公開管理を最優先にする
・閲覧者機能は閲覧とコメント参加に集中させる
・コメント機能はシンプルなスレッド方式に留める
・将来の査読や高度レビューへの拡張余地は残しつつ、現段階では実装しない
